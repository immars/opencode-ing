# 记忆系统

记忆系统的根目录是工作目录中的一个目录，比如`./.code-ing/memory`。

记忆系统都由文件构成。

记忆系统的组成，参照Level of detail的原则，按需加载。

## L0: 交流细节

* 详细的交流记录。这一层，需要存储跟所有用户通过飞书交流的原始消息。由插件程序维护。每次有新的用户消息，或者有agent的回复，都及时追加记录。
* 放在 `L0/{date}.md` 下面，每天一个文件。

## L1: 每天摘要

* 每天的交流记录的汇总。简要描述每天沟通了的话题内容。每天不超过500字节，尽量保留关键字信息的前提下，越简短越好。由agent自动从L0的内容中总结，放在L1中。
* 放在 `L1/{date}.md` 下面，每天一个文件。

## L2: 每周摘要

* 每周的交流记录的汇总。简要描述每周沟通了的话题内容。每周不超过500字节，尽量保留关键字信息的前提下，越简短越好。由agent自动从L1的内容中总结，放在L2中。
* 放在 `L2/{date}.md` 下面，每周一个文件。

## L9: 长期记忆

放在workspace的根目录。

* `SOUL.md`：agent的个性。当agent反思习得一些技能、习惯、工作方式的时候，由 agent 更新此内容。
* `PEOPLE.md`：人类用户的个性。跟agent交互的其他个体的用户画像。包含这个个体的关注点、经常做的事情、偏好。由 agent 更新。
* `TASK.md`: 当前来自人类用户的进行中的任务。记清楚什么时候，来自谁，任务的内容，和当前执行状态。task项的添加、删除 由agent 自动维护。
* `CRON.md`：定时任务。来自人类设置的定时任务。记清楚什么时候，来自谁，任务的内容。task项的添加、删除 由agent 自动维护。
* `CRON_SYS.md`：定时任务。系统定时任务。agent无法更改。
** 系统任务：每个小时，按照要求，汇总最新的 L1, L2 中的 md 文件。

# 组装逻辑

agent的工作有几种触发方式。code-ing 会往context中注入记忆信息。

* 飞书消息触发。注入：工作目录的根目录； L9中的SOUL.md，PEOPLE.md，TASK.md；L0中，最近60条消息；L1中最近3天的信息；L2中最近3周的信息。剩下的，让agent按需自己读取。
* 定时触发。注入：工作目录的根目录；L9中的 SOUL.md，PEOPLE.md，TASK.md；以及 CRON.md 和 CRON_SYS.md 中，按照当前触发的时间，应该激活的任务。

# 定时逻辑

`CRON.md`和 `CRON_SYS.md` 支持定时任务设置。支持类似crontab的时间设置格式，如：

```
# 定时任务1
* name: 任务1
* schedule: `*/30 * * * *`
* description: 去检查一下代码库
* completion: 检查完毕

# 定时任务2
* name: 任务2
* schedule: `0 0 * * *`
* descrption: 复盘一天的工作，维护记忆文件
* completion: 维护完成

```

code-ing提供内生的心跳机制。每半个小时（整点，即 xx:00 和 xx:30的时候），进行一次定时触发逻辑。
