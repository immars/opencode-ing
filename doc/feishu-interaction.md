# 飞书交互改进项

* 跟飞书的连接，需要有断连重连机制。每半个小时通过合适的api检查当前的连接是否正常；如果不正常，则主动重连。

* 每次plugin收到用户的消息以后，可以先给用户的消息标记一个表情包表示“正在处理”；然后当agent针对这个消息做了回复以后，则取消这个表情包标记。

* 在workspace内，持久化记忆所有有过交互的用户、群的id。这样做的目的是：可以主动往这些用户/群发一些消息，而不是只能被动回复。

* 每次消息返回给飞书用户的时候，加一个prettyfier美化环节，实现几个逻辑：
- 去掉一些系统可能附带的信息，比如<dcp-system-reminder>标记后面的信息
- 消息可能包含markdown的一些格式。看看是否有办法支持飞书消息这些格式的正确渲染
